LOS CAMBIOS NO SE VEN EN WWW.CALLSHIP.US HASTA QUE SUBAS server/public A GIT.

Railway tiene Root Directory = "server" y el build del server es "echo No build step".
Por tanto el frontend que ve la web es SOLO lo que está en server/public en Git.

PASOS cada vez que cambies el frontend (dialer) o quieras que se vea en producción:

  1. En la raíz del proyecto:
     npm run build

  2. Subir el build y los archivos tocados:
     git add server/public dialer/src server/src
     git status
     git commit -m "Deploy: Pendientes en menú, build actualizado"
     git push

IMPORTANTE: En "git status" DEBE aparecer server/public. Si no aparece, los cambios no se verán en la web.

Después del push: Railway desplegará solo. En www.callship.us haz Ctrl+F5.
Para comprobar que es el build nuevo: abre https://www.callship.us/build.txt

En Railway: Variables - ADMIN_USERS debe ser SOLO el usuario admin (ej: medinax6). No incluyas nicol2.
